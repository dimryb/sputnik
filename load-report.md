# Протокол нагрузочного тестирования

## Цель
Проверить стабильность и производительность эндпоинта `/health` микросервиса `sputnik` под нагрузкой.

## Методика
- Запущен сервис `sputnik` через Docker Compose (`docker-compose.e2e.yml`)
- Выполнено **100 последовательных HTTP-запросов** к `http://sputnik:8080/health`
- Использован легковесный контейнер `curlimages/curl` внутри той же Docker-сети
- Замерялось время полного ответа (`time_total`) для каждого запроса

## Результаты
Все 100 запросов завершились успешно с HTTP-статусом **200 OK**.  
Время отклика стабильно низкое:

- Минимальное: **0.32 мс**
- Максимальное: **0.78 мс**
- Среднее: **~0.35 мс**

Пример выборки:
```
Request 1: 0.000778s
Request 25: 0.000560s
Request 50: 0.000346s
Request 75: 0.000322s
Request 100: 0.000337s
```

Полный лог: [reports/load-report.txt](reports/load-report.txt)


Сервис `sputnik` демонстрирует высокую отзывчивость и стабильность даже при многократных обращениях.  
Эндпоинт `/health` пригоден для использования.
